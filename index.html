Hajj Crowd Monitoring and Control from Video Data

Previous Next
  • Demo: campuseat.herokuapp.com
  • Duration: Feb 2019 – June 2019

Huge crowds of people gather in Makkah for Hajj. Both Arafat and Muzadalifah lead to Mina and there are possible collision points in Mina. There is about a crowd flow of 2 million people in 2 hours at Mina. The number of pilgrims and the Hajj deaths due to catastrophes over the past years have increased overtime and no impactful steps have been taken to resolve this issue.

The current monitoring system makes use of cctv cameras, human operators and helicopters, but it has a lot of problems i.e. Crowd collisions, deaths, overcrowding and stampede. By using only have the cctv footage, we got the frames, applied a mask on it and used it to calculate the respective Crowd Count and Crowd Flow which will be used for further processing to approach the solution.

By using the direction of people they are moving it we form clusters and we get the velocity and count of the crowd in 8 directions.

This project could possibly mean that there would be a decrease in the number of catastrophes happening during the time of the Hajj and would possibly allow the Saudi Management to identify problems with the current infrastructure and would allow them the locations where and how the infrastructure can be improved.

The project was developed in Python using Numpy, Matplotlib, Pandas , CV2 , TensorFlow , Cuda and concepts of Machine Learning and Computer Vision.

SMS Spam Detection

Previous Next
  • Duration:Sept 2019 – Dec 2019
  • Website:github.com/HoshiHamza/SMS-Spam-Detection

It was a joint project of 5 people. It was developed in Python using Pandas, Numpy, Matplotlib, Scikit Learn and SciPy in addition to using concepts of Data Science and Machine Learning. A blogpost was created to present the project. It explores data consisting of tagged SMS messages collected over time. Data was first cleaned and then features were extracted from it before training.This showcases a predictive model that will help predict SMS messages as ham or spam in the future.

This model will help users in the following ways:

  • Spam messages will be marked automatically by the system, and hence it will save the user’s time.
  • It will prevent users from possible scams and frauds.
    • Details are provided in the blogpost given in the Github repository given in the link.
